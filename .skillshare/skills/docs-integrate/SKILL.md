# Documentation Integration Checklist

This checklist ensures every new page is correctly wired into the documentation site. The sidebar is **autogenerated** from `_category_.json` files and numeric filename prefixes — there is no `sidebars.js` to edit manually.

---

## Step 1: Assign the Correct Filename and Position

Files are sorted by their numeric prefix within each directory. Pick the next available number:

- `docs/guides/` — currently `01` through `08`; a new guide is `09-kebab-name.md`
- `docs/reference/` — currently `01` through `03`; a new reference page is `04-kebab-name.md`
- `docs/concepts/` — currently `01`; a new concept page is `02-kebab-name.md`

The filename prefix controls sidebar ordering. Choose a number that places the page logically in the reading sequence.

---

## Step 2: Add Correct Frontmatter

Every documentation page must have frontmatter with at minimum `id` and `title`:

```yaml
---
id: 09-guide-name
title: "How to <accomplish task>"
sidebar_label: "<Short Label>"
---
```

- `id` — must match the filename stem (without `.md`), e.g. `09-guide-name`
- `title` — the full page title shown in the browser tab and page header
- `sidebar_label` — optional; use it when `title` is too long for the sidebar

Do **not** add a `#` markdown heading that duplicates the `title` — frontmatter title is sufficient.

---

## Step 3: Update `docs/index.md` (If Warranted)

Add a link from the homepage if the new page is a top-level guide or reference that a new user would want to discover. Place it under the appropriate existing section in `docs/index.md`:

- New guides → link under the "Browse guides" section or quick-link list
- New reference pages → link under "Reference" or the closing navigation block

Do not add a homepage link for minor additions or supplementary content.

---

## Step 4: Cross-Reference Related Pages

Link the new page from (and to) related existing documentation:

- If the guide covers `JetStream`, add a "See also" or inline link in `docs/guides/03-jetstream.md`
- If the guide demonstrates a type in detail, add a reference link in the relevant reference page
- If the guide builds on a prerequisite guide, link the prerequisite in the Prerequisites section

Bidirectional links are preferred — both pages should reference each other.

---

## Step 5: Run mdoc Compilation

This step is **mandatory**. Run mdoc on the new file to verify all code examples compile and all internal links resolve:

```text
/c/Users/pieter/AppData/Local/Coursier/data/bin/sbt.bat "docs/mdoc --in docs/guides/NN-guide-name.md --out website/docs/guides/NN-guide-name.md"
```

**Success criterion:** Zero `[error]` lines. Warnings are acceptable.

If mdoc reports errors, fix them before proceeding. Do not commit a page with compilation errors.

---

## Quick Checklist

- [ ] Filename has correct numeric prefix for position in sequence
- [ ] Frontmatter has `id`, `title`, and optionally `sidebar_label`
- [ ] No duplicate `#` heading that repeats the frontmatter title
- [ ] `docs/index.md` updated if this is a prominent new page
- [ ] Related existing pages link to this new page
- [ ] This new page links back to related pages
- [ ] mdoc compiles with zero errors
