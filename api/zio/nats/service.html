trait NatsService

A running NATS microservice.

A running NATS microservice.

Obtained via zio.nats.Nats.service. The service is automatically stopped (with drain) when the enclosing Scope ends.

Provides access to the service's identity, health, and statistics — the same information that remote clients can query via ServiceDiscovery.

==Example==

ZIO.scoped {
 for {
   svc <- nats.service(ServiceConfig("calc", "1.0.0"), bound)
   _   <- Console.printLine(s"Service ${svc.name} started [${svc.id}]")
   _   <- ZIO.never
 } yield ()
}

enum QueueGroupPolicy

Controls queue group behaviour for a service endpoint.

Controls queue group behaviour for a service endpoint.

The NATS service framework uses queue groups for transparent load balancing: when multiple instances of the same service run, each request is delivered to exactly one instance.

  • QueueGroupPolicy.Default — use the standard service queue group "q".
  • QueueGroupPolicy.Disabled — disable queue grouping so every instance receives every request.
  • QueueGroupPolicy.Custom — use a caller-supplied queue group name.

trait ServiceDiscovery

Client for discovering NATS microservices running on the cluster.

Client for discovering NATS microservices running on the cluster.

Sends requests to the well-known $SRV.* subjects and collects responses from all running services (or a named subset). Uses jnats' io.nats.service.Discovery under the hood.

All methods block the calling thread until either the max-wait timeout expires or maxResults responses have been collected. Wrapped in ZIO.attemptBlocking so they run on the blocking thread pool.

==Example==

for {
 discovery <- ServiceDiscovery.make(maxWait = 3.seconds, maxResults = 50)
 services  <- discovery.ping()
 _         <- ZIO.foreach(services)(s => Console.printLine(s"${s.name} v${s.version} [${s.id}]"))
} yield ()

final class ServiceEndpoint[In, Err, Out](val name: String, val subject: Option[String] = ..., val queueGroup: QueueGroupPolicy = ..., val group: Option[ServiceGroup] = ..., val metadata: Map[String, String] = ...)(using val inCodec: NatsCodec[In], val outCodec: NatsCodec[Out], val errorMapper: ServiceErrorMapper[Err])

A declarative, typed descriptor for a NATS service endpoint.

A declarative, typed descriptor for a NATS service endpoint.

Captures the endpoint's name, optional subject override, queue group policy, optional group prefix, metadata, and — crucially — the input type, error type, and output type with their NatsCodec / ServiceErrorMapper instances. The descriptor is inert: it describes the shape of an endpoint but contains no handler logic.

To create a running endpoint, call implement or implementWithRequest to bind a handler function:

// Infallible endpoint (Err = Nothing)
val echo = ServiceEndpoint[String, String]("echo")
val bound = echo.implement(payload => ZIO.succeed(payload))

// Endpoint with a domain error type
given ServiceErrorMapper[UserError] with { ... }
val lookup = ServiceEndpoint[UserQuery, UserResponse]("lookup").withError[UserError]
val bound2 = lookup.implement(q => UserRepo.find(q))

Type parameters

Err

The handler error type. Use Nothing for infallible handlers. Use .withError[E] on a 2-type-param endpoint to set a domain error type.

In

The request payload type (decoded from bytes via NatsCodec[In]).

Out

The response payload type (encoded to bytes via NatsCodec[Out]).

trait ServiceErrorMapper[E]

Typeclass for converting handler error types to NATS service error responses.

Typeclass for converting handler error types to NATS service error responses.

NATS service errors are a (message, code) pair surfaced to the caller via the Nats-Service-Error and Nats-Service-Error-Code headers.

Built-in instances are provided for zio.nats.NatsError and String. Define your own for domain error types:

enum UserError:
 case NotFound(id: Int)
 case InvalidInput(msg: String)

given ServiceErrorMapper[UserError] with
 def toErrorResponse(e: UserError): (String, Int) = e match
   case UserError.NotFound(id)      => (s"User $id not found", 404)
   case UserError.InvalidInput(msg) => (msg, 400)
